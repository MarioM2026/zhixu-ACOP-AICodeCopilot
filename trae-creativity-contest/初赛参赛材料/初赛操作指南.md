## 三、完整发帖步骤

### 步骤 1：打开初赛专区

访问初赛专区：https://forum.trae.cn/c/38-category/40-category/40

### 步骤 2：发帖

1. 点击 **发帖** 按钮
2. 选择话题标签：**学习工作**
3. 填写标题：`学习工作赛道 · 知墟 - AI 编程助手统一观测与优化平台`
4. 复制 `初赛作品帖.md` 中的全部内容到正文
5. 点击附件上传按钮，选择 `知墟-初赛体验版.html`
6. 点击 **发布**

### 步骤 3：补充截图和 Session ID

发帖后，请补充以下内容（在帖子下方评论区补充）：

#### 3.1 添加开发过程截图（不少于 3 张）

请在 TRAE IDE 中截取以下关键步骤：

1. **截图 1**：项目初始化 + 技术选型对话
2. **截图 2**：规则引擎核心逻辑开发
3. **截图 3**：告警通知功能开发
4. **截图 4**：前端页面优化（可选）

#### 3.2 添加 Session ID（不少于 3 个）

Session ID 是 TRAE 每段对话任务的唯一标识。

**获取方式：**
1. 在 TRAE IDE 中打开关键开发对话
2. 双击对话标题或 URL 区域
3. 复制 Session ID（格式类似：`sess_xxx`）

**建议获取的 Session ID：**
- 项目初始化 + 技术选型
- 规则引擎核心逻辑实现
- 告警通知功能开发
- 前端页面优化

### 步骤 4：确认体验方式

初赛要求提供以下任一体验方式：

**方式一：本地运行（推荐）**

```bash
git clone https://github.com/your-username/ZhiXu-ACOP.git
cd ZhiXu-ACOP
npm install
npm run dev
```

**方式二：部署到 Vercel/Railway 等平台**

部署后将公开访问链接添加到帖子中。

**方式三：直接体验 HTML 文件**

评审可以直接在浏览器中打开 `知墟-初赛体验版.html` 体验核心功能。

## 四、初赛评审维度

根据大赛官方说明，专业评审按以下 4 个维度打分：

| 维度 | 说明 | 知墟的优势 |
|------|------|----------|
| **真实性** | 是否基于真实场景、真实问题 | ✅ 解决开发者实际痛点 |
| **创新性** | 是否提供独特的视角或解决方案 | ✅ "让 AI 越来越聪明"的反馈闭环 |
| **完成度** | 核心功能是否跑通、能否体验 | ✅ 四大核心模块完整实现 |
| **可展示性** | 能否清晰展示核心功能与目标场景 | ✅ 提供可体验 HTML 文件 |

## 五、TRAE 辅助开发说明

知墟项目使用 **TRAE IDE** 辅助开发完成，主要开发过程包括：

1. **项目初始化**：使用 TRAE 创建 React + TypeScript + Express 项目结构
2. **规则引擎实现**：TRAE 辅助设计规则引擎核心逻辑
3. **告警通知功能**：TRAE 辅助实现钉钉/邮件/Webhook 三通道
4. **前端页面优化**：TRAE 辅助优化 UI 和交互体验

**Session ID 获取位置：**
在 TRAE IDE 中，每个对话任务都有唯一的 Session ID。双击对话标题或地址栏即可复制。

## 六、常见问题

**Q：报名还没通过可以发初赛帖吗？**
A：不可以。初赛作品有效的前提是报名审核已通过。建议先确保报名通过。

**Q：可以同时提交多个 Demo 吗？**
A：可以提交多个 Demo，但晋级复赛时只取得分最高的 1 个作品。

**Q：截图必须 3 张以上吗？**
A：是的，官方要求不少于 3 张开发关键步骤截图。

**Q：Session ID 必须 3 个以上吗？**
A：是的，官方要求不少于 3 个关键任务对话的 Session ID。
