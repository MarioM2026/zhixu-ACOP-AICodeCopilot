## 一、Demo 简介

### 1.1 是什么

**知墟（ZhiXu）** 是一款 AI 编程助手统一观测与优化平台 Web 应用。

**一句话概括：** 知墟 = AI 编程助手的"体检中心 + 健康管家 + **进化引擎**"。

**核心价值主张：** 知墟的终极目标不是"监控 AI"，而是"让 AI 自我进化"。

### 1.2 面向谁

- **独立开发者**：日常使用 Trae/Claude Code/Cursor，希望优化 AI 使用效率
- **小团队技术负责人**：关注团队 AI 工具使用情况和 ROI
- **AI 编程学习者**：希望理解如何更好地使用 AI 编程工具

## 二、Demo 创作思路

### 2.1 灵感来源

在使用 Trae 和 Claude Code 进行日常开发的过程中，我发现：

1. **数据孤岛问题**：多个 AI 编程助手分别产生数据，Token 消耗、错误率等指标分散，无法统一查看。

2. **被动使用困境**：AI 产生错误后，下次可能还会犯同样的错误。缺乏反馈闭环，AI 不会从错误中学习。

3. **缺少量化依据**：Token 消耗了不少，但不知道花在哪些场景；代码反复调试，不知道是提示词问题还是上下文问题。

**核心洞察：** 如果有一个平台能像"AI 体检中心"一样，自动收集数据、用规则引擎发现问题，并将错误模式反馈给 AI —— 就能让 AI 越来越聪明。

### 2.2 想解决的问题

| 痛点 | 描述 | 解决方案 |
|------|------|---------|
| 数据分散 | 多工具数据无法统一查看 | 统一数据总线 + 可视化看板 |
| 缺少反馈闭环 | AI 重复犯错，无法自我进化 | 错误模式注入 → AI 上下文优化 |
| 成本不可控 | Token 消耗无法归因 | 实时监控 + 告警通知 |
| 被动等待问题 | 出了问题才发现 | 规则引擎主动预警 |

### 2.3 为什么做这个方向

**差异化定位：** 市面上的监控工具大多是"被动展示数据"，而知墟的核心是**主动执行 + 反馈闭环**——让 AI 从错误中学习，真正做到"越用越聪明"。

**技术可行性：** 基于 OpenTelemetry 标准协议，数据采集标准化；Node.js + React 技术栈成熟，MVP 可快速验证。

### 4.4 核心技术实现

**规则引擎（最核心）：**

```typescript
// 规则引擎核心逻辑
class RuleEngine {
  // 定时扫描事件数据
  async scanAndEvaluate(events: AICodeEvent[]) {
    for (const rule of this.rules) {
      if (rule.evaluate(events)) {
        await this.executeAction(rule);
      }
    }
  }

  // 执行动作：告警 / 清理上下文 / 注入提示
  async executeAction(rule: Rule) {
    switch (rule.action.type) {
      case 'send_alert':
        await this.alertService.send(rule.action);
        break;
      case 'clear_context':
        await this.traeAdapter.cleanupContext();
        break;
      case 'inject_prompt':
        await this.traeAdapter.injectPrompt(rule.action.prompt);
        break;
    }
  }
}
```

**反馈闭环流程：**

```
📊 采集数据 → 🔍 分析模式 → ⚙️ 执行规则 → 💡 注入提示 → 🧠 AI 进化
     ↑                                                              ↓
     └─────────────── 验证效果 ← 持续优化 ←───────────────────────┘
```

### 4.5 踩坑与解决

| 问题 | 解决方案 |
|------|---------|
| 前端 API 返回结构不一致 | 统一后端返回格式 `{ success, message, data }` |
| 告警测试失败 | 添加错误原因提示，区分成功/失败状态 |
| 规则动作执行链路未打通 | 规划下一阶段：executeAction 集成 alertService |
