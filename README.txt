【适用环境】
  - 操作系统: Windows 10 / Windows 11 （64 位）
  - 必需组件: Node.js v18.0 或更高版本（含 npm）
  - 推荐内存: 4 GB 或更多
  - 推荐磁盘: 至少 2 GB 可用空间（含依赖）

【快速上手（三步启动）】

  第 1 步：安装 Node.js
    如果尚未安装，请访问：
      https://nodejs.org/zh-cn
    下载并安装 LTS（长期支持）版本（推荐 v18 或 v20）
    安装完成后打开 CMD 或 PowerShell，输入：
      node --version
      npm --version
    如能显示版本号即安装成功。

  第 2 步：安装项目依赖
    双击运行本目录下的：
      【安装依赖.bat】
    首次运行会自动下载依赖包，需要 3-10 分钟
    （取决于网络速度，后续启动会非常快）

  第 3 步：启动平台
    双击运行本目录下的：
      【启动.bat】
    等待服务启动完成（约 30 秒）
    浏览器会自动打开管理界面

【访问地址】
  - 前端管理界面:  http://localhost:3000
  - 后端 API:      http://localhost:3001
  - API 健康检查:  http://localhost:3001/api/health

【停止服务】
  双击运行本目录下的：
    【停止.bat】

【常见问题】

  Q1: 双击 .bat 文件后一闪而过怎么办？
  A: 请右键以"管理员身份运行"。
     或者在当前目录按住 Shift + 右键，选择
     "在此处打开 PowerShell 窗口"，然后输入
     cmd /c 启动.bat 查看具体错误信息。

  Q2: npm install 卡住或失败怎么办？
  A: 尝试切换国内镜像源：
     在 PowerShell 中执行：
       npm config set registry https://registry.npmmirror.com
     然后重新运行【安装依赖.bat】

  Q3: 端口 3000 或 3001 被占用怎么办？
  A: 运行【停止.bat】可以自动释放端口
     或者手动查找并结束进程：
       netstat -ano | findstr :3001
       taskkill /F /PID <进程号>

  Q4: 启动后页面打不开怎么办？
  A: 请确认：
     1) 【启动.bat】窗口仍在运行（没有关闭）
     2) 浏览器地址是否正确: http://localhost:3000
     3) 是否在启动日志中看到 "listening on port" 字样
     4) 防火墙没有阻止 Node.js 访问网络

  Q5: 如何更新平台到最新版本？
  A: 1) 备份 data/ 目录（重要数据）
     2) 从 GitHub / Gitee 拉取最新代码
     3) 重新运行【安装依赖.bat】
     4) 双击【启动.bat】

【数据与配置】
  - 项目数据:     data/ 目录（事件、告警、规则等）
  - 运行日志:     logs/ 目录
  - 环境配置:     .env 文件（可选，可自定义端口等）
  - 告警配置:     通过管理界面设置（钉钉/邮件/Webhook）

【安全提示】
  1. 请勿将 .env 文件分享给他人（可能包含密钥）
  2. 生产环境请修改默认端口并启用 HTTPS
  3. 定期备份 data/ 目录下的重要数据
